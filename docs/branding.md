# Void OS Branding Guidelines

Use these rules to keep visuals and messaging consistent across code, docs, and website.

## Naming

- Public product name: **Void OS**
- Internal technical key names allowed: `VoidOS` (for registry keys and compatibility-sensitive paths)
- Repository name: `Void-OS`

## Voice and messaging

- Tone: direct, technical, user-respectful.
- Positioning: performance, privacy, and control.
- Avoid exaggerated claims or unverifiable benchmark promises.

## Visual direction

- Base palette: deep navy, silver/white gradients, subtle mid-blue accents.
- Avoid purple accents unless intentionally used for a specific campaign.
- Keep UI clean with minimal clutter and strong readability contrast.

## Link ownership checklist

When publishing releases, verify these all point to your own org/repo/domain:

- GitHub repository URLs
- Release links
- Issue/discussion links
- Website and docs links
- Support/Community links
